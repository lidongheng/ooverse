# 地图下钻方案

## 一、ECharts 地图下钻能力

ECharts 地图**支持下钻**，但需要自行实现逻辑，没有内置的「一键下钻」API。

### 实现思路

1. **准备多级地图 GeoJSON**：各级别地图需单独 GeoJSON 文件，如世界地图、中国地图、中国各省地图，通过 `echarts.registerMap` 注册。
2. **监听事件触发下钻**：通过 `chart.on('dblclick')` 监听双击，根据点击区域/标记判断下钻目标。
3. **切换地图**：在回调中 `setOption`，将 `geo.map` 切换为下级地图，并更新散点数据。
4. **返回上级**：提供返回按钮或右键事件，切换回上级地图。

### 示例代码结构

```javascript
// 注册多级地图
echarts.registerMap('world', worldGeoJson);
echarts.registerMap('china', chinaGeoJson);

// 双击散点下钻
chart.on('dblclick', 'series.scatter', (params) => {
  const name = params.name;
  if (name.startsWith('华北') || name.startsWith('华东') || /* ... */) {
    chart.setOption({
      geo: { map: 'china' },
      series: [{ data: chinaScatterData }],
    });
  }
});
```

## 二、本项目适配要点

结合当前 `HomeView.vue` 实现，下钻时需注意：

| 模块 | 说明 |
|------|------|
| **坐标偏移** | 世界地图有经度偏移（`shiftWorldGeoJSON`、`shiftLng`），下钻到中国地图时需关闭 |
| **引导线 + 标签** | `buildMarkersGraphic` 中 `convertToPixel('geo', coord)` 依赖当前 geo，切换地图后需重新构建 |
| **regionData 过滤** | 下钻到中国时只展示中国 Region，返回时恢复全部 |
| **visualMap 范围** | 下钻后数据子集 min/max 可能变化，需重新计算 |
| **中国地图 GeoJSON** | 需单独引入，注意合规性（可参考阿里 DataV） |

## 三、各大洲 GeoJSON 获取

ECharts 官方**不提供**各大洲 GeoJSON，需自行准备。

### 数据来源

| 来源 | 说明 |
|------|------|
| **阿里 DataV GeoAtlas** | https://datav.aliyun.com/portal/school/atlas/area_selector — 仅中国行政区划，无其他大洲 |
| **Natural Earth** | https://www.naturalearthdata.com/ — 全球矢量数据，可按大洲筛选导出 |
| **geojson-maps（GitHub）** | 如 `johan/world.geo.json` 等，提供各国 GeoJSON，可手动按大洲拆分 |
| **GeoJSON.io / MapShaper** | 在线编辑、裁剪 GeoJSON，可按属性筛选导出 |

### 从现有 world.geo.json 拆分

项目已有 `src/assets/geo/world.geo.json`，通常每个 feature 的 `properties` 中有大洲字段（如 `continent`、`region_un`），可按此拆分：

```javascript
const asiaFeatures = worldGeoJson.features.filter(
  f => f.properties.continent === 'Asia'
);
const asiaGeoJson = { type: 'FeatureCollection', features: asiaFeatures };
```

建议先检查 `world.geo.json` 的 feature properties 是否包含大洲分类字段，若有则无需额外下载。
