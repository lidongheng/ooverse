# Docker 容器调试手册

本文记录 ooverse 前端项目从构建镜像到停止容器的本地调试命令。

所有命令默认在项目根目录执行：

```bash
cd /Users/donghengli/Desktop/www/ooverse
```

## 1. 检查 Docker

确认 Docker Desktop 已启动：

```bash
docker version
```

正常情况下会同时显示 `Client` 和 `Server` 信息。

只显示 Client 或提示无法连接 Docker daemon，说明 Docker Desktop 没有正常启动。

## 2. 构建前端产物

构建 SIT 环境：

```bash
PROFILE=sit sh .cloudbuild/build.sh
```

构建 UAT 环境：

```bash
PROFILE=uat sh .cloudbuild/build.sh
```

构建生产环境：

```bash
PROFILE=prod sh .cloudbuild/build.sh
```

成功后，根目录应生成：

```text
ooverse/
├── cost-operation/
├── digital-map-cost/
├── ioc/
└── operation-map/
```

检查入口文件：

```bash
ls ooverse/*/index.html
```

## 3. 构建 Docker 镜像

```bash
docker build \
  --pull=false \
  -f .cloudbuild/Dockerfile \
  -t ooverse-web:local \
  .
```

参数说明：

- `--pull=false`：优先使用本地已有的 Nginx 基础镜像。
- `-f`：指定 Dockerfile 路径。
- `-t ooverse-web:local`：镜像名为 `ooverse-web`，标签为 `local`。
- 最后的 `.`：使用项目根目录作为 Docker 构建上下文。

查看镜像：

```bash
docker images ooverse-web
```

查看镜像详细信息：

```bash
docker image inspect ooverse-web:local
```

查看镜像架构：

```bash
docker image inspect ooverse-web:local \
  --format '{{.Architecture}}'
```

MacBook Air M4 本地构建通常显示：

```text
arm64
```

## 4. 创建并启动容器

```bash
docker run \
  -d \
  --name ooverse-web \
  -p 8080:80 \
  ooverse-web:local
```

参数说明：

- `-d`：在后台运行容器。
- `--name ooverse-web`：指定固定容器名，方便后续操作。
- `-p 8080:80`：把 Mac 的 8080 端口映射到容器的 80 端口。
- `ooverse-web:local`：使用前面构建的镜像。

如果没有指定 `--name`，Docker 会生成类似 `kind_taussig` 的随机名称。后续命令必须使用实际容器名或容器 ID。

## 5. 查看容器状态

只看正在运行的容器：

```bash
docker ps
```

查看全部容器，包括已经停止的容器：

```bash
docker ps -a
```

只查看 ooverse-web：

```bash
docker ps -a --filter name=ooverse-web
```

查看容器运行状态：

```bash
docker inspect ooverse-web \
  --format '{{.State.Status}}'
```

正常运行时输出：

```text
running
```

查看端口映射：

```bash
docker port ooverse-web
```

预期输出：

```text
80/tcp -> 0.0.0.0:8080
```

## 6. 访问四个项目

浏览器访问：

```text
http://localhost:8080/ooverse/cost-operation/
http://localhost:8080/ooverse/digital-map-cost/
http://localhost:8080/ooverse/ioc/
http://localhost:8080/ooverse/operation-map/
```

命令行检查：

```bash
curl -I http://localhost:8080/ooverse/cost-operation/
curl -I http://localhost:8080/ooverse/digital-map-cost/
curl -I http://localhost:8080/ooverse/ioc/
curl -I http://localhost:8080/ooverse/operation-map/
```

正常应返回：

```text
HTTP/1.1 200 OK
```

## 7. 查看容器日志

查看已有日志：

```bash
docker logs ooverse-web
```

持续查看日志：

```bash
docker logs -f ooverse-web
```

显示最后 100 行并持续跟踪：

```bash
docker logs \
  --tail 100 \
  -f \
  ooverse-web
```

按 `Control + C` 退出日志跟踪，不会停止容器。

如果容器使用的是随机名称，也可以使用容器 ID：

```bash
docker logs f5a35b9e001f
```

## 8. 进入容器

进入容器 Shell：

```bash
docker exec -it ooverse-web sh
```

进入后查看 Nginx 静态文件：

```sh
find /usr/share/nginx/html/ooverse \
  -maxdepth 2 \
  -name index.html
```

查看目录：

```sh
ls -la /usr/share/nginx/html/ooverse
```

退出容器：

```sh
exit
```

也可以不进入容器，直接执行命令：

```bash
docker exec ooverse-web \
  find /usr/share/nginx/html/ooverse \
  -maxdepth 2 \
  -name index.html
```

## 9. 检查 Nginx

检查 Nginx 配置语法：

```bash
docker exec ooverse-web nginx -t
```

查看完整 Nginx 配置：

```bash
docker exec ooverse-web nginx -T
```

查看 Nginx 进程：

```bash
docker exec ooverse-web ps
```

## 10. 停止、启动和重启

停止容器：

```bash
docker stop ooverse-web
```

启动已经停止的容器：

```bash
docker start ooverse-web
```

重启容器：

```bash
docker restart ooverse-web
```

停止后查看状态：

```bash
docker ps -a --filter name=ooverse-web
```

停止容器不会删除容器，也不会删除镜像。

## 11. 删除容器

先停止，再删除：

```bash
docker stop ooverse-web
docker rm ooverse-web
```

强制删除正在运行的调试容器：

```bash
docker rm -f ooverse-web
```

删除容器不会删除镜像。

## 12. 删除镜像

```bash
docker image rm ooverse-web:local
```

如果仍有容器依赖该镜像，需要先删除对应容器。

## 13. 修改代码后的重新部署

修改前端代码后，完整重新部署流程：

```bash
docker rm -f ooverse-web

PROFILE=sit sh .cloudbuild/build.sh

docker build \
  --pull=false \
  -f .cloudbuild/Dockerfile \
  -t ooverse-web:local \
  .

docker run \
  -d \
  --name ooverse-web \
  -p 8080:80 \
  ooverse-web:local
```

然后检查：

```bash
docker ps
docker logs ooverse-web
curl -I http://localhost:8080/ooverse/operation-map/
```

## 14. 常见问题

### No such container

错误：

```text
No such container: ooverse-web
```

先查看真实容器名称：

```bash
docker ps -a
```

如果容器名是随机名称，可以重命名：

```bash
docker rename kind_taussig ooverse-web
```

### 容器名称已经被占用

错误通常包含：

```text
Conflict. The container name "/ooverse-web" is already in use.
```

删除旧容器后重新创建：

```bash
docker rm -f ooverse-web
```

### 端口已经被占用

如果 8080 被占用，可以改用 8081：

```bash
docker run \
  -d \
  --name ooverse-web \
  -p 8081:80 \
  ooverse-web:local
```

访问地址相应改为：

```text
http://localhost:8081/ooverse/operation-map/
```

### 容器启动后立即停止

查看状态和日志：

```bash
docker ps -a --filter name=ooverse-web
docker logs ooverse-web
```

检查退出码：

```bash
docker inspect ooverse-web \
  --format '{{.State.ExitCode}}'
```

### 页面返回 404

确认访问地址包含 `/ooverse/<项目名>/`，然后检查容器内文件：

```bash
docker exec ooverse-web \
  find /usr/share/nginx/html/ooverse \
  -maxdepth 2 \
  -name index.html
```

如果缺少文件，需要重新执行前端构建和 Docker build。

### 页面还是旧版本

确认已经重新构建前端产物和镜像，并删除旧容器：

```bash
docker rm -f ooverse-web
PROFILE=sit sh .cloudbuild/build.sh
docker build --no-cache -f .cloudbuild/Dockerfile -t ooverse-web:local .
docker run -d --name ooverse-web -p 8080:80 ooverse-web:local
```

`--no-cache` 只用于确认缓存问题，日常构建通常不需要添加。
