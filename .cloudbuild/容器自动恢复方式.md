# Docker 容器自动恢复方式

本文说明 MacBook Air 合盖、重新开盖或 Docker Desktop 重启后，如何让 `ooverse-web` 容器自动恢复运行。

## Mac 合盖时会发生什么

MacBook Air 正常合盖后会进入睡眠状态：

- Docker Desktop 的 Linux 虚拟机会暂停。
- 容器不会继续执行任务。
- `localhost:8080` 暂时无法访问。
- Docker 镜像和容器不会因为睡眠被删除。

重新开盖并唤醒 Mac 后，Docker Desktop 通常会恢复，容器可能继续保持运行。

查看正在运行的容器：

```bash
docker ps
```

查看包括停止状态在内的全部容器：

```bash
docker ps -a
```

如果容器处于停止状态，可以手动启动：

```bash
docker start ooverse-web
```

## 为已有容器设置自动恢复

给已经创建的 `ooverse-web` 容器设置重启策略：

```bash
docker update \
  --restart unless-stopped \
  ooverse-web
```

检查重启策略：

```bash
docker inspect ooverse-web \
  --format '{{.HostConfig.RestartPolicy.Name}}'
```

预期输出：

```text
unless-stopped
```

## 创建容器时设置自动恢复

如果容器还没有创建，可以在 `docker run` 时直接设置：

```bash
docker run \
  -d \
  --name ooverse-web \
  --restart unless-stopped \
  -p 8080:80 \
  ooverse-web:local
```

参数说明：

- `-d`：让容器在后台运行。
- `--name ooverse-web`：设置固定容器名称。
- `--restart unless-stopped`：Docker Desktop 恢复后自动启动容器。
- `-p 8080:80`：把 Mac 的 8080 端口映射到容器的 80 端口。
- `ooverse-web:local`：使用本地构建的镜像。

## unless-stopped 的行为

`unless-stopped` 表示：

- Docker Desktop 重启后自动恢复容器。
- Mac 重启并启动 Docker Desktop 后自动恢复容器。
- Mac 从睡眠中恢复后，容器会随 Docker Desktop 恢复。
- 如果主动执行 `docker stop`，Docker 不会擅自重新启动该容器。

主动停止：

```bash
docker stop ooverse-web
```

再次手动启动：

```bash
docker start ooverse-web
```

## 开盖或重启后的检查步骤

检查 Docker Desktop 状态：

```bash
docker desktop status
```

检查容器状态：

```bash
docker ps -a --filter name=ooverse-web
```

查看最近 20 行日志：

```bash
docker logs \
  --tail 20 \
  ooverse-web
```

检查页面是否恢复：

```bash
curl -I http://localhost:8080/ooverse/operation-map/
```

正常应返回：

```text
HTTP/1.1 200 OK
```

## 容器没有自动恢复怎么办

先查看状态：

```bash
docker ps -a --filter name=ooverse-web
```

容器存在但已经停止时，执行：

```bash
docker start ooverse-web
```

容器不存在但镜像仍然存在时，检查镜像：

```bash
docker images ooverse-web
```

然后重新创建容器：

```bash
docker run \
  -d \
  --name ooverse-web \
  --restart unless-stopped \
  -p 8080:80 \
  ooverse-web:local
```

Mac 睡眠或 Docker Desktop 重启不会删除 `ooverse-web:local` 镜像，因此通常不需要重新执行 `docker build`。
